What does this webpage do?

The page can make a table out of IATI-published aid activity data: for instance from a file published by an aid agency about its activities in a particular country.

When aid agencies use the IATI standard to publish information about their work, this involves:

  • a computer language called XML
  • a schema – rules for arranging the information in the XML

The result is a document – or set of documents – difficult for humans to read directly, but systematic enough for computers. Computers can then be used to analyze, re-combine, and present the information in flexible ways.

Simplified example:

Data which in the XML schema looks something like this -

  • <iati-activity>
    • <title> Ababa water project</title>
    • <description> This project makes tube-wells with hand-pumps to provide public potable water sources in Ababa District</description>
  • </iati-activity>
  • <iati-activity>
    • <title> Health for all in Beeb</title>
    • <description> Primary health services in Beeb are expanded through recruitment and training of new staff </description>
  • </iati-activity>

- when put into the table is displayed like this:

title description
Ababa water project This project makes tube-wells with hand-pumps to provide public potable water sources in Ababa District
Health for all in Beeb Primary health services in Beeb are expanded through recruitment and training of new staff

The tabular format is easier for a human eye to scan, so as to take in the range of a group of activities, making comparisons and connections.

Why might that be useful?

It could be useful for...

  • ...finding details of a particular project if you already know which country it is in, and which aid agency will be reporting it. In that case you should be able to find it by scrolling through the generated table. A problem is that not all agencies report all of their activities, or report them with much helpful detail. So a second - perhaps stronger - use for our tabulator is...
  • ...seeing how much detail of aid activities is being reported by an agency, or aid agencies generally in a particular country of interest. At the same time, you may be...
  • ...getting a feel for the scope of a programme of an agency in a particular country - at least in some cases. You will be...
  • ...extending your familiarity with the characteristics of diverse projects, particularly where the agencies have provided links to project documents. Admittedly, this is in a minority of cases. In general, you should be...
  • ...gaining awareness of the size of the IATI undertaking but also the patchiness of the data and a sense of some of the reasons for this. For people with a special interest, one of the greatest benefits of this webpage may be helping with...
  • ...learning about the IATI reporting format . This tool keeps the user in closer sight of the source data - and all its problems - than most other tools for using IATI data. Still it provides enough readability to be meaningful.

Explain about source data

The options explained:

Fetch data from the IATI datastore

In this option, we send an automated request to the IATI web-server to provide us with its latest copy of a file. The process of transferring the file can take a minute or more if it is a large one. Sometimes the transfer fails because the IATI server is down. But usually it works nicely.

Note that this is not the absolutely most authentic source data. For more details on the creation and nature of of source data, click here.

The most authentic sources of IATI activity data are the activity files posted by reporting aid agencies on their own websites. The IATI Datastore uses these to update its database every 24 hours or so.
